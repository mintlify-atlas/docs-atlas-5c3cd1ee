The personas endpoints give you programmatic access to everything Polysona knows about a persona: the structured psychological profile, voice nuances, platform accounts, interview logs, and a scored QA simulation across 20 representative follower archetypes. All data is parsed at request time from PLOON-format Markdown files on disk — there is no caching layer.

---

## GET /api/personas

Returns all persona directories found under `personas/`, sorted alphabetically by ID. The `name` field is read from the `name:` key in each persona's `persona.md` file. If that file is missing, the directory name is used as a fallback.

```bash
curl http://localhost:3000/api/personas
```

**Example response**

```json
[
  { "id": "default", "name": "TestUser" },
  { "id": "techfounder", "name": "Alex Kim" }
]
```

<ResponseField name="id" type="string">
  The persona's directory name under `personas/`. Used as the `:id` path parameter for all other persona endpoints.
</ResponseField>

<ResponseField name="name" type="string">
  The value of the `name:` key from the persona's `persona.md`. Falls back to the directory name if the file is absent.
</ResponseField>

## GET /api/personas/:id

Returns the full parsed PLOON data for a single persona, combining all three source files: `persona.md`, `nuance.md`, and `accounts.md`. Each file is parsed independently and returned under its respective key. If a file does not exist in the persona directory, the corresponding key is returned as an empty object `{}`.

Returns `404` if the persona directory does not exist or is empty.

<ParamField path="id" type="string" required>
  The persona directory name (e.g. `default`). Must match an existing subdirectory under `personas/`.
</ParamField>

```bash
curl http://localhost:3000/api/personas/default
```

**Example response**

<ResponseField name="id" type="string">

<ResponseField name="accounts" type="object">
  Parsed contents of `accounts.md`. Contains role model references and virtual follower profiles. Returns `{}` if the file does not exist.
</ResponseField>

## GET /api/personas/:id/interview-log

Returns the parsed interview log entries from the `## interview-log` section of a persona's `persona.md`. Each entry corresponds to a date-stamped line in the PLOON source (lines starting with `~YYYY-MM-DD:`).

Returns `404` if the persona's `persona.md` file does not exist.

<ParamField path="id" type="string" required>
  The persona directory name.
</ParamField>

```bash
curl http://localhost:3000/api/personas/default/interview-log
```

**Example response**

```json
{
  "id": "default",
  "interviewLog": [
    {
      "date": "2026-03-29",
      "content": "McAdams Life Story: 삶의 전환점은 첫 번째 프로젝트 실패에서 재건으로의 여정이었다."
    },
    {
      "date": "2026-03-29",
      "content": "Laddering (+MI+ACT): 속도 -> 기회 포착 -> 무관련성에 대한 두려움 (terminal value: 자율성)."
    },
    {
      "date": "2026-03-29",
      "content": "Clean Language: 반복적인 은유 '돌파구 직전의 엔진 과열'."
    },
    {
      "date": "2026-03-29",
      "content": "Summary: 스토리 아크가 반응적 생존에서 의도적 저자로 이동; 원하는 단순함과 보호적 복잡성 사이의 모순이 남아있음."
    }
  ]
}
```

<ResponseField name="id" type="string">
  The persona ID as provided in the path parameter.
</ResponseField>

<ResponseField name="interviewLog" type="array">
  Array of log entries parsed from the `## interview-log` section of `persona.md`.

  <Expandable title="interviewLog entry">
    <ResponseField name="date" type="string">
      ISO date string in `YYYY-MM-DD` format, parsed from the `~YYYY-MM-DD:` prefix in the source file.
    </ResponseField>
    <ResponseField name="content" type="string">
      The text content of the log entry, following the date prefix on that line.
    </ResponseField>
  </Expandable>
</ResponseField>

## GET /api/personas/:id/qa-simulation

Runs a scored QA simulation for a persona against all 20 follower archetypes. Each archetype receives a score across five dimensions. Scores are **deterministic** — the same persona ID, follower ID, dimension, and content name always produce the same score, making results reproducible and diffable.

Returns `404` if none of the three PLOON files (`persona.md`, `nuance.md`, `accounts.md`) exist for the given persona ID.

<ParamField path="id" type="string" required>
  The persona directory name.
</ParamField>

<ParamField query="content" type="string">
  Optional content name string. Including this parameter makes scores specific to that piece of content — useful when simulating how a particular post would perform across follower types. If omitted, scores reflect the persona alone.
</ParamField>

```bash
# Simulate without specific content
curl http://localhost:3000/api/personas/default/qa-simulation

# Simulate for a specific content piece
curl "http://localhost:3000/api/personas/default/qa-simulation?content=my-draft-post"
```

**Example response**

```json
{
  "personaId": "default",
  "contentName": "my-draft-post",
  "followers": [
    {
      "id": "f04",
      "label": "startup founder",
      "age": 29,
      "gender": "M",
      "occupation": "founder",
      "scores": {
        "hook": 82,
        "empathy": 71,
        "share": 88,
        "cta": 65,
        "platform_fit": 79
      },
      "total": 385,
      "isTop5": true
    },
    {
      "id": "f02",
      "label": "30s male developer",
      "age": 32,
      "gender": "M",
      "occupation": "dev",
      "scores": {
        "hook": 74,
        "empathy": 68,
        "share": 77,
        "cta": 59,
        "platform_fit": 83
      },
      "total": 361,
      "isTop5": true
    }
  ],
  "top5": [ "...top 5 follower objects..." ],
  "generatedAt": "2026-03-29T12:00:00.000Z"
}
```

<ResponseField name="personaId" type="string">
  The persona ID used in the simulation.
</ResponseField>

<ResponseField name="contentName" type="string">
  The `content` query parameter value, or an empty string if not provided.
</ResponseField>

<ResponseField name="followers" type="array">
  All 20 follower archetypes with scores, sorted by `total` descending (highest total first).

  <Expandable title="follower object">
    <ResponseField name="id" type="string">
      Archetype identifier (`f01` through `f20`).
    </ResponseField>
    <ResponseField name="label" type="string">
      Human-readable archetype description (e.g. `"30s male developer"`).
    </ResponseField>
    <ResponseField name="age" type="number">
      Representative age for the archetype.
    </ResponseField>
    <ResponseField name="gender" type="string">
      `"F"`, `"M"`, or `"NB"`.
    </ResponseField>
    <ResponseField name="occupation" type="string">
      Short occupation tag (e.g. `"dev"`, `"founder"`, `"marketing"`).
    </ResponseField>
    <ResponseField name="scores" type="object">
      Five scored dimensions, each an integer in the range **40–95**.

      <Expandable title="scores object">
        <ResponseField name="hook" type="number">
          How effectively the content grabs this archetype's attention (40–95).
        </ResponseField>
        <ResponseField name="empathy" type="number">
          How well the content resonates emotionally with this archetype (40–95).
        </ResponseField>
        <ResponseField name="share" type="number">
          Likelihood that this archetype would share or reshare the content (40–95).
        </ResponseField>
        <ResponseField name="cta" type="number">
          Likelihood that this archetype would act on a call to action (40–95).
        </ResponseField>
        <ResponseField name="platform_fit" type="number">
          How well the content format fits this archetype's typical platform usage (40–95).
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="total" type="number">
      Sum of all five dimension scores. Maximum possible value is 475 (5 × 95).
    </ResponseField>
    <ResponseField name="isTop5" type="boolean">
      `true` if this follower ranks in the top 5 by total score for this simulation run.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="top5" type="array">
  The five follower objects with the highest `total` scores. Same shape as individual entries in `followers`.
</ResponseField>

<ResponseField name="generatedAt" type="string">
  ISO 8601 timestamp of when the simulation was run.
</ResponseField>
